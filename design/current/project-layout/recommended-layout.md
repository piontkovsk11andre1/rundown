# Recommended layout

The standard directory layout for a rundown-managed project. Configurable via `config.json` `workspace` keys but the defaults match the conventions every command assumes.

```
<repo-root>/
├── .rundown/                # rundown config dir
│   ├── config.json
│   ├── locale.json
│   ├── vars.json
│   ├── templates/
│   │   ├── run.md
│   │   ├── verify.md
│   │   ├── repair.md
│   │   └── ...
│   ├── memory/              # see memory-layout.md
│   ├── locales/             # message catalogs per locale
│   └── runs/                # runtime artifacts; see runtime-artifacts.md
│
├── design/
│   ├── current/             # editable design (this directory)
│   │   ├── README.md
│   │   ├── overview/
│   │   ├── architecture/
│   │   └── ...
│   └── rev.<N>/             # immutable snapshots created by `design release`
│
├── migrations/
│   ├── 1. <Title>.md        # plan + checkboxes for revision N
│   ├── 2. <Title>.md
│   └── ...
│
├── specs/                   # acceptance specs / test plans (optional)
│   └── ...
│
├── implementation/          # the source workspace (optional)
│   ├── package.json
│   ├── src/
│   └── ...
│
├── AGENTS.md                # auto-generated by `start`
└── README.md
```

## Why these names

- **`.rundown/`** — discovered by upward walk; mirrors `.git/` ergonomics.
- **`design/current/`** — separation between the working design (writable) and snapshots (`rev.<N>/`, immutable). See [../lifecycle/design-revisions.md](../lifecycle/design-revisions.md).
- **`migrations/`** — numbered files, sortable; each file captures planner-authored intent for a single revision delta and is consumed by lifecycle tooling.
- **`specs/`** — optional location for acceptance specs referenced by `verify` and `migrate`.
- **`implementation/`** — optional, for monorepo-style projects where rundown manages design but is decoupled from the source workspace. The `workspace.implementation` config field lets users point elsewhere.

## Customization

`config.json`:

```json
{
  "workspace": {
    "design":         "docs/design",
    "migrations":     "docs/migrations",
    "specs":          "docs/specs",
    "implementation": "."
  }
}
```

Custom paths are resolved relative to the directory containing the `.rundown/`. Missing custom directories are created lazily by commands that write them.

## .gitignore guidance

Recommended entries (also documented in [../execution/trace-and-artifacts.md](../execution/trace-and-artifacts.md) and [../ci/concurrency-and-safety.md](../ci/concurrency-and-safety.md)):

```
.rundown/runs/**
.rundown/worker-health.json
*.lock
```

Everything else inside `.rundown/` is meant to be tracked.

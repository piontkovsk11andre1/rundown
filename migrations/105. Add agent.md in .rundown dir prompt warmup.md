migrate: Add agent.md in .rundown dir (prompt) that will be a part of rundown (no args) warmup.

## Intent

Root no-argument rundown help should include a dedicated warmup prompt file, `.rundown/agent.md`, as first-class prompt input before interactive help guidance. This formalizes warmup behavior as a repository-local template instead of an implicit/default-only behavior.

The goal is to keep root help startup deterministic, configurable, and consistent with existing template-driven architecture.

## Current behavior and implementation context

The codebase already has partial support for this concept:

- Default warmup template exists in `src/domain/defaults.ts` as `DEFAULT_AGENT_TEMPLATE`.
- Project initialization already writes `.rundown/agent.md` in `src/application/init-project.ts`.
- Template loading already resolves `agent.md` from config dir with fallback in `src/application/project-templates.ts`.
- Documentation describes `.rundown/agent.md` as warmup for root no-arg live help (`docs/templates.md`, `docs/cli.md`).

This migration should align prompt assembly behavior with those existing surfaces so runtime behavior, init scaffolding, and docs all match.

## Feature specification

Add or confirm `agent.md` as an explicit warmup input in root no-argument rundown flow:

1. Root no-arg help prompt construction prepends `agent.md` warmup content before `help.md` guidance.
2. If `.rundown/agent.md` is missing, unreadable, or empty, fallback remains `DEFAULT_AGENT_TEMPLATE`.
3. This behavior applies specifically to root no-arg help mode and should not alter execute/plan/research/verify prompt contracts.
4. Warmup content remains repository-local and versionable under `.rundown/`.

## Prompt and worker contract alignment

When documenting or normalizing examples, maintain OpenCode conventions already used in the repo:

- Deterministic flows (`run`, `plan`, `research`, `reverify`) use `opencode run`.
- Interactive `discuss` uses base `opencode`.
- Prompt transport remains file-first (`$file` + `$bootstrap`) for large contexts and Windows/PowerShell reliability.

Although this migration targets root no-arg warmup, any related docs/examples should not regress these conventions.

## Scope boundaries

In scope:

- Root no-arg warmup composition behavior around `.rundown/agent.md`.
- Clear fallback semantics to built-in defaults.
- Consistency across init scaffolding, template loading, and root-help prompt assembly.
- Documentation clarity for template role and no-arg behavior.

Out of scope:

- Changing deterministic command worker wiring (`run`/`plan`/`research`/`reverify`).
- Changing discuss worker mode behavior.
- Redesigning non-root prompt templates (`execute.md`, `verify.md`, etc.).
- Introducing new task decomposition behavior or TODO insertion changes.

## Compatibility and constraints

- Preserve existing repos that already rely on default warmup text without a local `agent.md`.
- Do not break `rundown init` idempotence or config-dir override behavior.
- Keep behavior stable in linked workspace mode; template lookup should continue using resolved config directory rules.
- Do not alter checkbox/task-file semantics; this migration is prompt composition and documentation alignment.

## Edge cases to account for

- `.rundown/agent.md` absent: fallback to default warmup.
- `.rundown/agent.md` present but empty/whitespace: treat as empty and continue safely (fallback or no-op per existing loader semantics).
- Root no-arg invocation with warnings: preserve canonical startup ordering and keep warmup integrated without duplicating intro text.
- Custom config dir (`--config-dir`) and discovered config dir should both resolve `agent.md` consistently.

migrate: add plan --loop mode with dedicated loop-planning template

We need a boolean `--loop` option for `plan` (flag only, no numeric value) that switches planner prompting from `plan.md` to `plan-loop.md`.

The loop-specific template must guide generation of bounded loop workflows that use:

- `get:` to discover an iterable set of values/items,
- `end:` to terminate when no values are found (or another explicit deterministic stop rule),
- `for:` to iterate those values and run per-item implementation/review tasks from prompt context.

Also cover commit behavior for loop execution paths (`--commit`) if any gap still exists.

## Intent

Introduce a dedicated planning mode for loop-oriented TODO authoring so users can request loop plans explicitly instead of relying on generic planner heuristics.

## Current implementation context

- Plan CLI options are wired in `src/presentation/cli.ts` and normalized in `src/presentation/cli-command-actions.ts`.
- Plan orchestration and prompt rendering are in `src/application/plan-task.ts`.
- Template loading/fallbacks are in `src/application/project-templates.ts`.
- Built-in templates and planning feature reference content are in `src/domain/defaults.ts`.
- Init scaffolding for `.rundown/*` templates is in `src/application/init-project.ts`.
- Docs for plan/template surfaces live in `docs/cli-plan.md` and `docs/templates.md`.
- Existing loop command already uses run-like option wiring (`configureRunLikeCommandOptions(loopCommand)`), so `--commit` may already be available, but this migration should verify and document it clearly.

## Required behavior

1. Add `plan --loop` as a boolean mode switch.
2. In loop mode, resolve planner template from `.rundown/plan-loop.md` (with built-in fallback when missing).
3. Keep normal `plan.md` behavior unchanged when `--loop` is not set.
4. Ensure loop-mode guidance strongly favors `get:` + `for:` + `end:` composition and deterministic stop conditions.
5. Clarify and test loop `--commit` support and docs (implement only if a real gap exists).

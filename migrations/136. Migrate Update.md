# Plan: Migration 136 — Migrate Flow Redesign

## TL;DR

Redesign `rundown migrate` from a one-shot interactive proposal picker into a plan-style convergence loop. Eliminate context (N.4) as a concept. Replace per-numbered satellites with singleton `migrations/Backlog.md`. `migrate up` generates numbered `N.1 Snapshot.md` at batch end. `migrate down` optionally pushes removed migrations to Backlog. Remove all dead subcommands.

---

## Decisions Made

- **Context (N.4) dropped** — snapshot is the single source of truth for predicted state
- **Backlog = `migrations/Backlog.md`** — singleton, never numbered, persistent across cycles
- **Snapshot = `N.1 Snapshot.md`** — numbered, permanent, one per `migrate up` batch, previous stay as history
- **Convergence = DONE keyword** from AI, detected in code, loop exits cleanly
- **migrate down → Backlog** — removed migration names/content pushed to Backlog.md (disableable via `--no-backlog`)
- **Removed subcommands**: `snapshot`, `backlog`, `context`, `user-experience`, `user-session`
- **Plan document is transient** — not written to filesystem; used only to drive file creation within a single loop iteration
- **Discussion of predicted state** — handled by normal `rundown` agent, which can update Backlog.md

## Phase 3 — migrate-task.ts Core Rewrite

**`src/application/migrate-task.ts`**

### `MigrateAction` type
Remove: `"snapshot"`, `"backlog"`, `"context"`, `"user-experience"`, `"user-session"`  
Keep: `"up"`, `"down"`

### New `runMigrateLoop()` — replaces `generateNextMigration()`
The main no-args behavior becomes a loop:
```
loop:
  scanLoop:
    ask AI (migrate plan template): "what migrations are needed?"
    if output is "DONE": break outer loop
    parse proposed names from output
    if no new names: break scanLoop (converged on plan)
    add new names to planned list
    repeat scanLoop

  create migration files for all planned names (formatMigrationFilename)
  for each new file: run explore enrichment

  if --confirm: pause with prompt "Files created. Edit if needed. Continue?"

  call runMigrateUp() → generates N.1 Snapshot.md at batch end

  clear planned list, loop back
```

### `runMigrateUp()` — replaces/modifies `migrate up`
- Iterate existing migration files in order
- For each: run AI prediction task via `runTask` (like `run` does one by one)
- At batch end: call `generateSatellite({ satelliteType: "snapshot" })` with new snapshot template
- Template vars include: `{{latestSnapshot}}` (previous), `{{newMigrations}}` (content of migrations run in this batch)

### `runMigrateDown()` — modifies `migrate down`
- Delete last N migration files
- Unless `--no-backlog`: append removed migration names/seeds to `migrations/Backlog.md`
- Delete `N.1 Snapshot.md` satellites at positions > new endpoint
- Call `runMigrateUp()` to regenerate snapshot for new endpoint

### Remove entirely
- `generateNextMigration()` (old one-shot proposal with interactive picker)
- `runUserSession()`
- `parseProposals()` / `selectProposal()` (interactive picker)
- All calls to `generateSatellite()` for types: `context`, `backlog`, `user-experience`

### New `parseProposedMigrationNames(stdout)` 
Replaces `parseProposals()`. Parses simple list of kebab-case names from AI output. Returns `null` if output is `DONE`.

### Template vars update in `buildTemplateVars()`
- Remove: `latestContext`, `latestBacklog` vars
- Add: `latestSnapshot` (from `state.latestSnapshot`), `backlog` (content of `Backlog.md` if exists), `newMigrations` (for snapshot template)

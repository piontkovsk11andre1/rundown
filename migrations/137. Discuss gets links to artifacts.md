The core asymmetry: `discuss --run <id>` is fully artifact-aware (sees execution phases, verify verdicts, repair history). `discuss <mdfile>` (source mode — selects the next unchecked task) is completely blind to prior run attempts and acts as if the task was never run.

Fix this by making source-mode discuss discover and surface previous `run` attempts for the source file. The `--run` finished-run path is already correct and stays untouched.

Additionally, the default discuss prompt currently encourages proactive task rewriting. It should be conversational-first — no edits unless the user explicitly asks.

## Current behavior

`discuss` has two modes:

- **Source mode** (`discuss <mdfile>`): selects the next unchecked task, renders `DEFAULT_DISCUSS_TEMPLATE`, knows nothing about prior runs.
- **Finished-run mode** (`discuss --run <id|latest>`): resolves artifact run, scans phases via `scanRunPhases()`, renders `DEFAULT_DISCUSS_FINISHED_TEMPLATE` with full artifact context.

From the current implementation:

- `src/application/discuss-task.ts` — finished-run path already calls `scanRunPhases()` and builds `FinishedRunPromptContext` with phase dirs, verify verdicts, repair history, missing-log notes. Source-mode path calls only `resolveTaskContext()` and renders discuss template with task/file/source vars — no artifact lookup.
- `src/domain/defaults.ts` — `DEFAULT_DISCUSS_TEMPLATE` contains `"You may modify the source Markdown task text..."` which makes proactive editing the default behavior.
- `docs/cli-discuss.md` — currently implies the agent may edit task text during discussion.
- `artifactStore.listSaved()` / `listFailed()` — already available on the port; source-mode discuss simply never calls them.

## What needs to change

### 1. Related-run discovery in source mode (`src/application/discuss-task.ts`)

After `resolveTaskContext()` returns the selected task in source mode:

1. Call `artifactStore.listSaved(configDir)` and `artifactStore.listFailed(configDir)`.
2. Filter to entries where `commandName === "run"` AND `task?.file` matches the selected task's source file path. Normalize both sides via `pathOperations.resolve()` to handle relative vs. absolute. Fallback: if `task?.file` is absent, match on `task?.source` basename.
3. Merge saved + failed, sort by `startedAt` descending, cap at 5.
4. For each candidate, call the existing `scanRunPhases()` helper to get phase detail (exit code, verify verdict) — handle missing/deleted run dirs gracefully (mark as "artifacts unavailable").
5. Add a new helper `buildRelatedRunsSummary(candidates, fs, pathOps): string` alongside existing `formatPhaseSummary`. Output: formatted table with run ID, status, timestamp, run dir, brief phase outcome. When no candidates: `"No previous run attempts found for this task."`.
6. Pass `relatedRunsSummary` as an additional var to `renderDiscussPrompt()`.

### 2. Template vars + prompt enrichment (`src/domain/defaults.ts`)

Add a `## Related run history` section to `DEFAULT_DISCUSS_TEMPLATE` rendering `{{relatedRunsSummary}}`, placed before `## Task context`.

### 3. Prompt contract correction (`src/domain/defaults.ts`, `docs/cli-discuss.md`)

In `DEFAULT_DISCUSS_TEMPLATE`, replace the permissive-edit wording with a conversational-first contract:

- Primary role: analyze the task, answer questions, refer to prior run history.
- Do **not** edit source Markdown by default.
- Only edit when the user explicitly requests it (rewrite, split, add sub-items, clarify scope, etc.).
- Still must not change checkbox state.
- Still must not perform implementation work.

Update `docs/cli-discuss.md` to document: source mode now shows prior run context, discuss is non-mutating by default, edits happen only on explicit user request.

### 4. Tests (`__tests__/application/discuss-task.test.ts`)

Add test cases:

- Source-mode discuss prompt includes `relatedRunsSummary` var when prior `run` attempts exist for the source file.
- Source-mode discuss prompt includes "No previous run attempts" text when `listSaved`/`listFailed` return nothing matching.
- Existing `--run` finished-discuss tests pass unchanged (no regression).
- Updated `DEFAULT_DISCUSS_TEMPLATE` does not contain old permissive-edit wording.

## Out of scope

- The `--run` finished-run discuss path — already correct, stays untouched.
- Changing `run`/`verify`/`repair` execution semantics.
- Auto-migrating old artifacts or changing artifact retention policy.
- Checkbox completion ownership (already runner-managed, rollback already enforced).

## Edge cases

- **Missing artifacts**: runs whose phase dirs are gone → label as "artifacts unavailable", not an error.
- **Moved/deleted task files**: fallback to `task.source` basename matching so historical context is still surfaced.
- **Multiple tasks in same file**: match on `task.file` + task identity (text/line) when available to avoid surfacing runs for a different task in the same file.
- **No prior runs**: surface `"No previous run attempts found for this task."` — not an error, just context.
- **Config-dir differences**: use active `configDir` for all `artifactStore` calls; paths in summary use that same root.

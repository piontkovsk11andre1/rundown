## Bug

`tsc --noEmit` fails in `implementation/`:

```
src/infrastructure/adapters/fs-artifact-store.ts(52,7): error TS2322: Type 'RuntimePhaseHandle' is not assignable to type 'ArtifactPhaseHandle'.
  Types of property 'phase' are incompatible.
    Type 'RuntimePhase' is not assignable to type 'ArtifactStorePhase'.
      Type '"translate"' is not assignable to type 'ArtifactStorePhase'.
```

## Root cause

`RuntimePhase` (in `src/infrastructure/runtime-artifacts.ts`) includes `"translate"`, but the domain port `ArtifactStorePhase` (in `src/domain/ports/artifact-store.ts`) does not. `translate` was added at the runtime/worker level (see `src/application/translate-task.ts` passing `artifactPhase: "translate"`, `src/domain/worker-config.ts`, `src/domain/ports/worker-executor-port.ts`), but the domain port and adapter mapping were never updated.

The adapter `createFsArtifactStore.beginPhase` returns the runtime handle directly (cast is structural through `toRuntimePhase`), so the inferred return type still carries `RuntimePhase` — and the domain consumer types this as `ArtifactPhaseHandle` whose `phase` is the narrower `ArtifactStorePhase`.

## Fix

Add `"translate"` to the domain `ArtifactStorePhase` union so the port matches the runtime reality. This is a pure additive change — no behavior changes, no migration of existing artifacts, no template/CLI changes.

Also tighten the existing `worker-executor-port.ts` `artifactPhase?` ad-hoc string union to reuse `ArtifactStorePhase` so the two stay in sync going forward (currently it duplicates the list).

## Implementation boundaries

- `src/domain/ports/artifact-store.ts` — add `"translate"` to `ArtifactStorePhase`.
- `src/domain/ports/worker-executor-port.ts` — replace the inline `artifactPhase?` union with `ArtifactStorePhase` (import from `./artifact-store.js`); drop the redundant local list.
- No changes to `runtime-artifacts.ts`, no changes to adapter, no changes to application code.

## Out of scope

- Renaming `ArtifactStorePhase` / `RuntimePhase` to one canonical type.
- Restructuring the runtime-artifact module.
- Anything related to the recent move of application code into `implementation/`.

## Edge cases

- Existing JSON metadata on disk that recorded `phase: "translate"` already validates against the new union — no migration needed.
- `worker-executor-port.ts` consumers that previously accepted only the smaller inline union now accept the full `ArtifactStorePhase`. Verify no narrowing logic depended on the old list (grep for `artifactPhase ===`).

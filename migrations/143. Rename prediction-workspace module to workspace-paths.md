# 143. Rename Prediction Workspace Paths Module To Workspace Paths

## Problem

The module at `implementation/src/application/prediction-workspace-paths.ts` is named with the word "prediction" because, historically, the workspace it describes is the *prediction loop* workspace ‚Äî the directories rundown reads/writes during prediction work (`design/`, `specs/`, `migrations/`).

Migration 144 introduces a new bucket whose name is also `prediction` (the predicted-implementation tree, sibling to `implementation/`). Two unrelated meanings of "prediction" then collide in the codebase:

- The module name `prediction-workspace-paths`, the type names `PredictionWorkspaceDirectories` / `PredictionWorkspacePlacementMap` / `PredictionWorkspacePaths` / `PredictionWorkspaceBucket` / `PredictionWorkspacePlacement`, the constants `DEFAULT_PREDICTION_WORKSPACE_*` and `PREDICTION_WORKSPACE_PLACEMENTS`, and the functions `resolvePredictionWorkspace*` ‚Äî all use "Prediction" to mean "the workspace tracked by the prediction loop".
- A *new bucket field* named `prediction` (lowercase) inside those same types, meaning "the predicted-implementation directory".

This is confusing and untenable long-term. The fix is to rename the module and all its symbols to drop the `Prediction` prefix entirely. The remaining word ‚Äî `Workspace` ‚Äî is precise: this module is about the rundown-tracked workspace directories (regardless of whether they belong to the prediction loop or not).

This migration is a **pure mechanical rename**. No semantics change, no field added, no test logic changes. After this migration, migration 144 will introduce the new `prediction` bucket without naming collision.

## Model

After this migration, in `implementation/src/application/`:

- The file is named `workspace-paths.ts` (no "prediction" in the path).
- All exported and internal symbols replace `PredictionWorkspace`/`PREDICTION_WORKSPACE` with `Workspace`/`WORKSPACE`:

  | Before | After |
  |---|---|
  | `PredictionWorkspaceDirectories` | `WorkspaceDirectories` |
  | `PredictionWorkspaceBucket` | `WorkspaceBucket` |
  | `PredictionWorkspacePlacement` | `WorkspacePlacement` |
  | `PredictionWorkspacePlacementMap` | `WorkspacePlacementMap` |
  | `PredictionWorkspacePaths` | `WorkspacePaths` |
  | `PredictionWorkspaceConfig` | `WorkspaceConfig` |
  | `PREDICTION_WORKSPACE_PLACEMENTS` | `WORKSPACE_PLACEMENTS` |
  | `DEFAULT_PREDICTION_WORKSPACE_DIRECTORIES` | `DEFAULT_WORKSPACE_DIRECTORIES` |
  | `DEFAULT_PREDICTION_WORKSPACE_PLACEMENT` | `DEFAULT_WORKSPACE_PLACEMENT` |
  | `resolvePredictionWorkspaceDirectories` | `resolveWorkspaceDirectories` |
  | `resolvePredictionWorkspacePlacement` | `resolveWorkspacePlacement` |
  | `resolvePredictionWorkspacePath` | `resolveWorkspacePath` |
  | `resolvePredictionWorkspacePaths` | `resolveWorkspacePaths` |
  | `resolvePredictionWorkspaceConfig` | `resolveWorkspaceConfig` |

- All callers update their `import` statements: `from "./prediction-workspace-paths.js"` ‚Üí `from "./workspace-paths.js"`, and the imported symbol names are updated.
- The test file `implementation/__tests__/application/prediction-workspace-paths.test.ts` is renamed to `workspace-paths.test.ts` and its imports/symbols updated.
- No type field is added or removed. No function signature changes beyond the symbol names. No test assertion semantics change.

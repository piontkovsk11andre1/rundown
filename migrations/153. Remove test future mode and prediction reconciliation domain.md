# 153. Remove Test Future Mode And Prediction Reconciliation Domain

## Problem

After 151 (no `migrate up`) and 152 (no `migrate down` / backlog), the only remaining consumer of the prediction-reconciliation pipeline is `rundown test --future [n]`. Per the user's decision, prediction moves to a separate project; rundown's `test` command must evaluate assertions only against the live workspace.

This migration deletes:

- The `--future [n]` flag of `rundown test` ([implementation/src/presentation/cli.ts](implementation/src/presentation/cli.ts#L510)).
- The future-mode codepath in [test-specs.ts](implementation/src/application/test-specs.ts), including `FutureScope` construction, `latestSnapshot` / `latestContext` / `workspacePredictionPath` injection into the verify template's variables, and the prediction-tree composition that reads `prediction/` and migration files.
- The prediction-tree test template added in [templates-loader.ts](implementation/src/infrastructure/templates-loader.ts#L46) for future-mode.
- The entire [implementation/src/domain/prediction-reconciliation.ts](implementation/src/domain/prediction-reconciliation.ts) module, since 151/152 already removed its other consumers.
- The entire [implementation/src/application/prediction-workspace-paths.ts](implementation/src/application/prediction-workspace-paths.ts) module, used only by the future-mode test path and by the migrate-up flow that 151 removed.
- Any planner-template reference to `{{workspacePredictionPath}}` whose only purpose was to direct agents to the predicted tree. The runtime variable itself stays available (other commands like `start` still scaffold/reference the directory), but default templates stop pointing the agent at it.

After this migration `rundown test [source]` only does materialised mode against the current workspace.

## Model

After this migration:

- `rundown test --future` errors as an unknown option.
- `runTestSpecs` (or whatever the entry function is called in `test-specs.ts`) has a single mode: read assertion specs from `--dir` (default `specs/`) and verify each against the live workspace via the standard worker chain.
- `TestContext` has no `latestSnapshot`, `latestContext`, `latestContextPath`, `futureScope`, or any prediction-related field.
- `prediction-reconciliation.ts` does not exist.
- `prediction-workspace-paths.ts` does not exist.
- The verify template's `TemplateVars` no longer carries `latestSnapshot`, `latestContext`, or `workspacePredictionPath`. Default templates that referenced them are trimmed to a single instruction stream that only depends on the live workspace.
- The `runtime-workspace-context.ts` `workspacePredictionPath` runtime variable stays exported (used by `start` scaffolding, agent docs, and config defaults) — but no built-in template, no migrate flow, and no test flow consumes it.
- The default `migrate.md` planner template no longer says "Read the current prediction tree at `{{workspacePredictionPath}}` directly" (introduced by 150). The planner now sees only the design + migration history; predicted-tree reading is the new project's concern.

## Out of scope

- Removing the `workspacePredictionPath` runtime variable. The `start` command, project scaffolds, and CI workflows reference the path. The variable remains available even though no built-in template uses it post-pivot.
- Removing the `prediction/` workspace bucket directory itself or any user data on disk. Rundown stops touching it; users delete it themselves.
- Renaming `test-specs.ts` to reflect the simpler scope.
- Reshaping the `design/current/prediction/` tree. That's 154.
- Adding a programmatic API surface for the new prediction project. Deferred per decision.
- Localisation propagation of the modified default templates to non-English locales. Same policy as 150.

## Edge cases

- **A user invokes `rundown test --future`.** Commander returns "unknown option".
- **A workspace has a `prediction/` directory left over from `migrate up` runs.** Rundown ignores it. `rundown test specs/` evaluates only live workspace files.
- **A user-customised `test-verify.md` still references `{{latestSnapshot}}` or `{{workspacePredictionPath}}`.** Renderer leaves unknown placeholders as-is (per existing behaviour). No deprecation warning.
- **`design-context.ts` is now used only by `migrate` and `discuss`.** That's fine — keep the helpers those callsites need; delete only helpers that have no surviving consumer.
- **A test asserts on `runtime-workspace-context.ts` exporting `workspacePredictionPath`.** Keep that assertion; the variable stays.

Remove the `rundown:` prefix feature entirely. No backward compatibility, no migration path.

## Problem statement

The parser currently recognizes a `rundown:` prefix on checkbox task text to classify delegated rundown tasks. This prefix-based routing is unnecessary complexity. We want to delete the feature completely — remove the prefix constant, the parser classification, the `isRundownTask`/`rundownArgs` fields, all execution branches that consume them, and all tests that assert on them.

## What to remove

### Parser layer (`src/domain/parser.ts`)

- `RUNDOWN_PREFIX` constant.
- `isRundownTask` and `rundownArgs` fields on the `Task` type.
- The prefix-matching branch in `walkForTasks` that sets those fields.

### Execution and orchestration

- `src/application/run-task-iteration.ts`: `validateRundownTaskArgs` and any branch gated on `isRundownTask`.
- `src/application/task-execution-dispatch.ts`: `executeRundownTask` function, target validation, recursion/self-target guard, lifecycle output specific to rundown tasks.
- `src/application/dry-run-dispatch.ts`: delegated-rundown rendering and argument forwarding paths.
- `src/domain/run-options.ts`: `isRundownTask` bypass for worker-command requirements.

### Delegation module (`src/application/rundown-delegation.ts`)

- Remove entirely, or gut any logic that only existed to support the `rundown:` prefix flow. If the module has no remaining purpose, delete the file.

### Tests

- `__tests__/domain/parser.test.ts`: remove all `rundownArgs`/`isRundownTask` assertions.
- `__tests__/application/run-task-iteration-modules.test.ts`: remove delegated-args validation tests.
- `__tests__/application/rundown-delegation.test.ts`: remove or delete entirely.
- `__tests__/integration/cli.test.ts`: remove any test cases exercising `rundown:` prefix behavior.
- Any other test files asserting on rundown task detection, execution, or dry-run rendering.

## Scope

### In scope

- Delete all code implementing the `rundown:` prefix feature.
- Delete all tests for the feature.
- Clean up any types, interfaces, or options that only existed for this feature.

### Out of scope

- Inline CLI (`cli:`) prefix — unchanged.
- Other prefixes (memory capture, verify directives) — unchanged.
- CLI commands (`run`, `make`, `do`) — unchanged.
- Artifact formats, run IDs, trace schema — unchanged unless they reference removed fields.

## Acceptance criteria

- `RUNDOWN_PREFIX` constant no longer exists.
- `isRundownTask` and `rundownArgs` no longer exist on the `Task` type.
- No code path checks for or dispatches on rundown task classification.
- All tests pass with the feature fully removed.
- A task whose text happens to start with `rundown` is treated as a plain task.

# Research with TODO items

## Context

The `rundown research` command enriches markdown documents with context and detail. After the worker produces updated content, the result passes through constraint validation in `research-task.ts` (around line 530).

Currently, `listIntroducedUncheckedTodos()` compares unchecked `- [ ]` lines before and after research. When new unchecked TODOs are detected, the entire update is **rejected as a fatal error**: the original document is restored and the process exits with code 1. This is too aggressive — models occasionally inject TODO items despite being told not to, and discarding the entire enriched output is wasteful.

### Detection mechanism

`parseUncheckedTodoLines()` in `src/domain/todo-lines.ts` operates on **raw text lines** using the regex `/^\s*[-*+]\s+\[ \]\s+\S/`. It does **not** use the markdown AST, so `- [ ]` lines inside fenced code blocks (` ``` `) are currently matched the same as real TODO items. This must be accounted for.

### Desired behavior

Instead of rejecting the entire research output when new `- [ ]` TODO items appear:

1. **Strip** the introduced unchecked TODO lines from the updated document.
2. **Emit a warning** (not error) indicating the model did not follow instructions.
3. **Keep** the rest of the enriched content and write it to disk.
4. **Preserve** `- [ ]` lines that exist inside fenced code blocks — they are illustrative content, not real tasks.

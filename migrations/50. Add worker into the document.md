# Add `cli-args:` group-level directive to the document parser

Allow defining a `cli-args:` directive at the group level so it applies arguments to all sibling `cli:` tasks within that group, rather than repeating `--worker` (or other args) on every line.

## Feature specification

The `cli-args:` directive is a non-checkbox list item that carries a string value (the arguments to append). When nested `cli:` tasks are parsed, these arguments are appended to each `cli:` task's `cliCommand`. Non-CLI checkbox tasks under the same directive are unaffected.

Example usage:
```markdown
- cli-args: --worker opencode
  - [ ] cli: npm run build
  - [ ] cli: npm test
```
Both CLI tasks would execute with `--worker opencode` appended.

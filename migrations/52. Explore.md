# Add `explore` command — research + plan alias

Introduce an `explore` command that sequences `research` followed by `plan` on the same Markdown document. This is a convenience alias similar to how `make` composes create → research → plan, but for existing documents that already have content and just need enrichment and task synthesis in one step.

## Feature specification

`rundown explore <markdown-file>` runs research on the given file, then — if research succeeds — runs plan on the same file. All shared worker/runtime options are forwarded to both phases. Plan-specific options (`--scan-count`, `--deep`, `--max-items`) are forwarded only to the plan phase.

- **Exit codes**: Returns the exit code of the first failing phase, or `0` if both succeed.
- **Fail-fast**: If research fails, plan is skipped entirely.

## Scope

### In scope

- New `explore` CLI command that composes research + plan sequentially.
- Command action factory in `cli-command-actions.ts` following the `make` composition pattern.
- Command registration in `cli.ts` with research options + plan-specific options.
- Documentation update in `docs/cli.md`.
- Integration test verifying the sequential composition.

### Out of scope

- New application use case or domain types — this is purely a CLI-layer composition.
- Changes to `create-app.ts` or the `App` type — reuses existing `researchTask` and `planTask`.

## Implementation tasks

### Command action (`src/presentation/cli-command-actions.ts`)

- [x] Add `createExploreCommandAction({ getApp })` factory function that:
  1. Validates exactly one existing markdown file via `resolveResearchMarkdownFile()` (same validation as research)
  2. Builds shared worker/runtime options via `resolveSharedWorkerRuntimeOptions()`
  3. Calls `app.researchTask(request)` with the resolved file and shared options
  4. If research returns `0`, calls `app.planTask(request)` with the same file, shared options, and plan-specific options (`scanCount`, `deep`, `maxItems`)
  5. Returns the exit code of the first failing phase, or `0` if both succeed
- [x] Follow the `createMakeCommandAction` composition pattern (sequential delegation with fail-fast)

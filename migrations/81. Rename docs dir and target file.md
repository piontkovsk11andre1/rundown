Rename prediction design paths from `docs`/`Design.md` to `design`/`Target.md` across the project workflow.

## Intent

Align naming with the updated mental model:

- Directory: `design/` (replaces `docs/` for prediction workspace content)
- Current draft path: `design/current/Target.md` (replaces `docs/current/Design.md`)

This migration should update runtime resolution, command behavior, scaffolding, and user-facing docs while preserving compatibility for existing projects during transition.

## Scope

- Update prediction/design workspace defaults and path resolution to prefer `design/**` and `Target.md`.
- Keep legacy fallback support for existing `docs/**` and `Design.md` repositories.
- Ensure revision workflows still operate with immutable snapshots and deterministic ordering under the renamed directory.
- Update CLI help and documentation text to reflect the new canonical names.

## Compatibility constraints

- Existing projects that only contain `docs/current/Design.md` must continue to work.
- Existing projects that only contain root `Design.md` must continue to work where legacy fallback is already supported.
- New scaffolding should create only the new canonical structure unless explicit compatibility flags are required.

## Acceptance criteria

- Canonical workspace naming uses `design/current/Target.md` for new flows.
- Revision discovery/save/diff flows operate on `design/rev.*` and remain deterministic.
- Migration/test/research context resolution prefers `design/**` first, then falls back to legacy paths.
- CLI and docs reference the new names consistently while documenting fallback behavior.
- Existing tests pass, and updated coverage asserts both new canonical and legacy fallback behavior.

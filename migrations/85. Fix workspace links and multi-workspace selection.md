Fix linked-workspace behavior for `start`, support multiple workspace-link records, and add explicit workspace selection for path-sensitive commands.

## Intent

Current workspace-link behavior is inverted in some scenarios and does not scale when a source is linked to more than one workspace. This migration corrects link creation semantics and introduces explicit selection when ambiguity exists.

From epic feedback:

- In a directory with rundown, link files were created in the wrong place.
- Target workspace did not receive expected metadata.
- Multiple workspace links must be supported.
- Commands like migration/docs should accept `--workspace <dir>` when needed.

## Product goal

- `start` writes link metadata in the correct source/target locations.
- Link metadata can represent multiple workspace records.
- Path-sensitive commands resolve the effective workspace deterministically.
- When multiple records exist and no default is defined, CLI requires explicit workspace selection.

## Scope

- Correct link creation/update flow in `start`.
- Define and implement multi-record schema for workspace links.
- Add `--workspace <dir>` selection support to relevant commands (at least migration/docs flows).
- Add clear ambiguity handling and diagnostics when multiple workspace records are present.

## Behavior constraints

- Existing single-link repositories remain backward compatible.
- Relative vs absolute path normalization is deterministic and cross-platform safe.
- Broken/stale links are handled gracefully with actionable errors.
- Linked-workspace resolution remains explicit in command messaging (invocation dir vs effective workspace dir).

## Acceptance criteria

- `start` creates/updates link metadata in the intended source and target locations.
- Workspace link files support multiple records with deterministic lookup behavior.
- `migrate`/`docs` commands accept `--workspace <dir>` and resolve context against that workspace.
- If multiple records exist and no default is available, CLI lists candidates and requires explicit selection.
- Legacy single-record link files still function.

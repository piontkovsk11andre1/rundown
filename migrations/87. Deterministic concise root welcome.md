Define a deterministic, concise welcome message for the root `rundown` command and require agents to emit it first.

## Intent

Current root-command startup can produce random/freeform opening text from the agent. This creates inconsistency and weakens UX predictability.

We want a short, stable first response that appears before any additional guidance.

## Product goal

- Root `rundown` starts with one canonical concise welcome message.
- Agent output order is deterministic: welcome first, then contextual help or follow-up.
- Startup tone remains helpful without verbose/random phrasing.

## Scope

- Define canonical welcome copy (single source of truth).
- Update root-command startup flow to always emit that message first.
- Update agent/system prompt contract for root help mode to prioritize this first-response rule.
- Keep downstream command behavior unchanged.

## Message requirements

- Short (1-2 lines).
- Stable wording (no randomized variants).
- Mentions primary next actions succinctly (for example: plan/explore/run/help).
- Suitable for TUI and non-interactive logs.

## Behavior constraints

- Welcome appears exactly once at session start for root command invocation.
- It should not repeat on every subsequent internal turn unless a new root session starts.
- Existing exit codes and command semantics remain unchanged.
- If startup includes warnings/errors, welcome still appears first unless startup fails before runtime initialization.

## Acceptance criteria

- Running bare `rundown` consistently prints the same concise greeting first.
- Agent no longer emits random intro text before the canonical greeting.
- Help/discovery guidance still follows after the greeting.
- Docs/prompts describe the first-response requirement clearly.

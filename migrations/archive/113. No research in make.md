Argument --raw as rundown make --raw <args> that skip research and use seed to plan, alternative --skip-research

## Intent

Extend `rundown make` so users can bypass the research phase and go directly from seed creation to planning, matching existing skip-phase behavior already supported by `rundown query --skip-research`.

## Current behavior and context

`make` currently executes a strict two-phase bootstrap in `src/presentation/cli-command-actions.ts`:

1. create seed markdown file (`createSeedMarkdownFile`)
2. run `researchTask`
3. run `planTask` if research succeeds

The command wiring in `src/presentation/cli.ts` for `make` does not currently expose either `--skip-research` or `--raw`.

Related precedent already exists in `query`:

- CLI option: `--skip-research` in `src/presentation/cli.ts`
- normalized option forwarding in `createQueryCommandAction`
- use-case branching in `src/application/query-task.ts`:
  - normal path: phase 1 research, phase 2 plan, phase 3 execute
  - skip path: emits skip-phase info and starts at plan

This existing pattern is the canonical implementation reference for skip-phase semantics and user-facing messaging.

## Proposed CLI contract for `make`

Support skip-research for `make` with one canonical flag and one compatibility alias:

- Canonical: `--skip-research`
- Alias: `--raw` (same behavior)

Expected usage examples:

- `rundown make --skip-research "<seed-text>" "<markdown-file>" -- <worker>`
- `rundown make --raw "<seed-text>" "<markdown-file>" -- <worker>`

If both are provided, behavior is still a single boolean `true` (no conflict error needed).

## Behavioral specification

When skip-research is enabled for `make`:

- file creation still occurs first (unchanged)
- research phase is not executed
- plan phase runs against the seed file content directly
- phase logging reflects skip behavior (for parity with query UX)
- existing fail-fast semantics remain:
  - if seed file creation fails, command exits non-zero
  - if plan fails, command exits non-zero
- dry-run / print-prompt continue to function consistently with current orchestration conventions

When skip-research is disabled (default), current behavior remains unchanged.

## Compatibility and edge cases

- Preserve current positional argument validation (`resolveMakeMarkdownFile`) and markdown extension constraints.
- Preserve worker resolution rules (`--worker` vs separator `-- <command>`).
- Preserve non-overwrite behavior for existing target files.
- Avoid introducing a new mode enum; this is a boolean phase-toggle only.
- Ensure parity in terminology across commands (`skip research`, `start from planning`) to avoid user confusion between `make` and `query`.

## Acceptance criteria outline

- `rundown make` accepts `--skip-research` and `--raw`.
- Either flag causes make to skip research and execute plan after file creation.
- Default behavior without either flag remains unchanged (`create -> research -> plan`).
- Logs/messages clearly indicate when research is skipped.
- `docs/cli-make.md` reflects new option(s) and behavior.
- Automated tests cover forwarding and orchestration branching.
- No regression in existing make input validation or runtime option handling.

## Out of scope

- Changing `explore`, `do`, or `research` command semantics.
- Altering planning algorithm, scan/deep behavior, or task decomposition logic.
- Introducing new execution modes beyond existing `wait` handling for make.

# Plan: Fast Mode Prefix (`fast:` / `raw:`)

## TL;DR
Add `fast:` and `raw:` task prefixes that suppress verification on individual tasks — the inverse of `verify:`. Implementation follows the existing legacy intent path (same as `verify:` / `memory:`): detect via regex in `classifyTaskIntent`, strip prefix, suppress `shouldVerify` in `resolveIterationVerificationMode`. No changes to `BUILTIN_TOOLS` or dispatch — task executes via the default worker path with verification disabled.

## Architecture Decision
Use the **legacy intent path only** (not prefix chain / BUILTIN_TOOLS registration). Rationale:
- Registering as a handler in BUILTIN_TOOLS would produce a `"tool-handled"` chain result that short-circuits worker execution (line 258 of task-execution-dispatch.ts). Would require dispatch changes to handle `skipExecution: false` on tool-handled chains.
- The intent path is simpler: classify → strip prefix → set intent → iteration-mode suppresses verify. Default worker execution proceeds normally.
- This is consistent with how `verify:` is handled at the intent level (regex + intent → behavior in iteration-mode).

## Steps

### Phase 1: Domain — Intent Classification

1. **Add `"fast-execution"` to `TaskIntent` type** in `src/domain/task-intent.ts` (line 6)
   - Extend union: `"verify-only" | "memory-capture" | "tool-expansion" | "fast-execution" | "execute-and-verify"`

2. **Add fast prefix regex and detection** in `src/domain/task-intent.ts`
   - Add `const FAST_EXECUTION_PREFIX = /^(fast|raw)\s*:/i;` after `MEMORY_CAPTURE_PREFIX` (line 33)
   - In `classifyTaskIntent()`, add detection block after memory-capture check and before tool-expansion check (~line 106):
     - Match `FAST_EXECUTION_PREFIX`, strip prefix, return `{ intent: "fast-execution", reason: "explicit fast marker", normalizedTaskText: <stripped payload>, hasEmptyPayload: <bool> }`

### Phase 2: Domain — Parser Directive Propagation

3. **Add parser patterns** in `src/domain/parser.ts` (~lines 107-111)
   - Add `const FAST_DIRECTIVE_PATTERN = /^(?:fast|raw)\s*:\s*$/i;` (for `- fast:` parent items with no payload)
   - Add `const FAST_TASK_PREFIX_PATTERN = /^(?:fast|raw)\s*:/i;` (for prefix detection in `isIntentClassifiedPrefixTaskText`)

4. **Add fast directive detection** in `parseDirectiveParent()` (~line 381)
   - After the verify check: `if (FAST_DIRECTIVE_PATTERN.test(text)) { return { intent: "fast-execution" }; }`

5. **Add fast prefix to `isIntentClassifiedPrefixTaskText()`** (~line 352)
   - Add `|| FAST_TASK_PREFIX_PATTERN.test(normalized)` to the existing verify/memory check

### Phase 3: Application — Verification Suppression

6. **Suppress `shouldVerify` for fast-execution** in `src/application/iteration-mode.ts` `resolveIterationVerificationMode()` (~line 59)
   - Add: `const isFastExecution = taskIntent.intent === "fast-execution";`
   - Change: `const shouldVerify = (configuredShouldVerify || onlyVerify) && !isFastExecution;`
   - Add info message when fast execution is detected and verification is suppressed

7. **Handle empty fast payload** in `src/application/run-task-iteration.ts` (~line 249, after tool-expansion empty check)
   - Add early return/skip when `taskIntentDecision.intent === "fast-execution" && taskIntentDecision.hasEmptyPayload`
   - Emit warning: "Fast task has no payload text; skipping."

### Phase 4: Tests

8. **Unit tests in `__tests__/domain/task-intent.test.ts`**
   - `fast:` prefix → intent `"fast-execution"`, normalizedTaskText stripped
   - `raw:` prefix → same behavior (alias)
   - Case-insensitive matching: `FAST:`, `Raw :`, `fAsT:\t`
   - Preserves normalizedTaskText trimming
   - Empty payload detection: `fast:` → hasEmptyPayload true
   - Does not false-match: `"fast forward these docs"` → default intent
   - Precedence: `verify:` still wins when it appears first

9. **Parser tests in `__tests__/domain/parser.test.ts`**
   - `- fast:` parent directive propagates `intent: "fast-execution"` to child checkboxes
   - `- raw:` parent directive propagates similarly
   - `isIntentClassifiedPrefixTaskText` returns true for `fast: X`

10. **Application tests in `__tests__/application/run-task-iteration-modules.test.ts`**
    - Fast-execution intent suppresses `shouldVerify` even when `configuredShouldVerify` is true
    - Empty fast payload skips execution with warning

11. **Integration tests in `__tests__/integration/cli.test.ts`**
    - `run` with `fast:` prefixed task executes without verification
    - `run` with `raw:` prefixed task executes without verification

## Decisions
- **Legacy intent path only** (no BUILTIN_TOOLS registration) — avoids dispatch complexity for `tool-handled` + `skipExecution: false`
- **`fast` and `raw` as aliases** — both prefixes supported per migration spec
- **Order in classifyTaskIntent**: verify → memory → fast → tool-expansion → default
- **Excluded**: No prefix chain / BUILTIN_TOOLS registration (can be added later if needed for composability with other prefixes)

# 51. Error in stdout (cli command run)

## Background

The CLI output system routes `ApplicationOutputEvent` kinds to terminal streams (stdout/stderr) via `cliOutputPort` in `src/presentation/output-port.ts`, and records stream metadata in the JSONL log via `resolveLogStream()` in `src/presentation/logged-output-port.ts`.

Currently, `warn` events and `group-end` failure events are incorrectly sent to stdout instead of stderr. Per UNIX convention, diagnostics, warnings, and error indicators should go to stderr so that stdout contains only primary program output.

## Affected code

- `src/presentation/output-port.ts:422` â€” `warn` case uses `console.log` (stdout)
- `src/presentation/output-port.ts:394-407` â€” `group-end` uses `console.log` for both success and failure
- `src/presentation/logged-output-port.ts:127-143` â€” `resolveLogStream()` maps `warn` and `group-end` (failure) to `"stdout"`

## Stream routing table (current vs correct)

| Event kind             | Current stream | Correct stream |
|------------------------|---------------|----------------|
| `warn`                 | stdout        | **stderr**     |
| `group-end` (failure)  | stdout        | **stderr**     |
| `group-end` (success)  | stdout        | stdout         |
| `error`                | stderr        | stderr         |
| `info`                 | stdout        | stdout         |

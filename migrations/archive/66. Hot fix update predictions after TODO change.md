Add a hot-fix reconciliation feature for predicted migrations/snapshots when the task stream changes mid-execution.

Problem: during long migration runs, we sometimes inject a manual fix into an earlier task. That can invalidate downstream AI predictions (next migrations, expected snapshot/state). We need a recovery step that re-evaluates and reconciles the remaining predicted plan with the new reality.

Desired behavior:

- Detect when edits/fixes to TODO/migration content make future predictions stale.
- Ask AI to re-resolve the remaining sequence from the current state (not from original assumptions).
- Update pending predicted items/snapshots so execution can continue consistently.
- Preserve already completed migrations as historical fact; only reconcile future/pending parts.

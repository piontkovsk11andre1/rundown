Remove animated terminal output from `init` and eliminate animated text output patterns across the CLI.

## Intent

CLI output should be deterministic, script-friendly, and readable in both interactive and non-interactive environments. Animated text effects currently add noise and can interfere with logs, snapshots, and tooling.

This migration removes animation behavior from `rundown init` and establishes a project-wide non-animated output baseline.

## Product goal

- `init` output is plain, stable text with no frame-based or typed animation.
- Other commands avoid animated text patterns by default.
- UX remains clear through structured static messaging rather than animation.

## Scope

- Remove animation codepaths from `init` command flow.
- Audit and remove remaining animated text usage in CLI presentation/output utilities.
- Keep message content useful, but render it as immediate static output.

## Behavior constraints

- Output ordering must remain deterministic.
- No spinner/typing/frame animation should be required for command clarity.
- Command success/failure semantics and exit codes must remain unchanged.
- If any optional animation toggle exists, default must be non-animated and docs must reflect that.

## Acceptance criteria

- Running `rundown init` produces non-animated output only.
- No CLI command relies on animated text for core messaging.
- Existing command behavior and exit codes remain intact.
- Documentation/help no longer imply animated CLI text behavior where removed.
